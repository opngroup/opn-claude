# PR 작성 스킬

## Purpose

현재 브랜치의 변경 사항을 분석하여 프로젝트 PR 템플릿에 맞는 PR 설명을 자동 생성합니다.

## When to Activate

- "pr 작성해줘", "pr 작성", "write pr" 등의 키워드가 감지될 때

## Workflow

1. `git log main..HEAD --oneline`으로 현재 브랜치의 커밋 목록 확인
2. `git diff main...HEAD`로 전체 변경 사항 확인
3. 변경 내용을 분석하여 아래 템플릿에 맞게 PR 설명 작성

## PR 템플릿

```markdown
### PR 설명

<!-- 변경된 주요 기능·화면·API 등을 "한 줄 요약" 으로 작성합니다. -->

- 설명

---

### 주요 변경 사항

<!-- 주요 변경 사항 설명을 입력해 주세요 -->

- 변경사항

---

### 테스트 시나리오

<!-- 유닛 테스트, E2E, 인수 테스트 링크(노션 PRD)등을 입력해 주세요 -->

- 시나리오

---

### 고려한 사항

<!-- 암묵적인 요구사항, 기술적인 고려, 보안 이슈, 라이브러리 호환성 및 관리 , UXUI, 마케팅 트래킹 등을 입력해 주세요 -->

- 고려사항

---

### 참고

<!-- JIRA 티켓, Figma 링크, 노션 링크, 슬랙 링크, 그외 참고할만한 링크들을 입력해 주세요 -->

- 참고 링크
```

## 작성 규칙

- 한국어로 작성
- 각 섹션에 실제 변경 내용 기반으로 구체적으로 작성
- 테스트 시나리오는 실제 검증 가능한 단계로 작성
- 고려한 사항에는 기술적 결정의 이유를 포함
- 참고 섹션에는 관련 문서 링크가 있으면 포함

## Notes

- main 브랜치 대비 변경 사항을 기준으로 작성
- 커밋 메시지와 코드 diff를 모두 참고하여 정확한 내용 작성
